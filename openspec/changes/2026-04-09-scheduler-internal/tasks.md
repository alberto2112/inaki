## Phase 6: SchedulerService

- [ ] T6.1 Create `core/domain/services/scheduler_service.py` — `SchedulerService(repo, dispatch: SchedulerDispatchPorts, config: SchedulerConfig)`; `start()`: call `repo.seed_builtin(BUILTIN_CONSOLIDATE_MEMORY)` → `_handle_missed_on_startup()` → start async loop task; `stop()`: cancel loop task, await; `invalidate()`: sets `asyncio.Event` `_wake`; main loop uses `asyncio.wait_for(wake.wait(), timeout=sleep_for)` where `sleep_for = min(60, max(0, next_due.next_run - now))`; `_handle_missed_on_startup()`: fetch `list_due_pending(now)`, oneshot → `update_status(MISSED)` + `save_log`, recurrent → recompute `next_run` via `croniter.croniter(schedule).get_next()`; `_execute_task(task)`: set RUNNING, retry loop 0..max_retries writing one TaskLog per attempt, on success call `_finalize_task`, on max retries exceeded → FAILED; `_finalize_task`: oneshot success → COMPLETED+enabled=0, recurrent countdown=0 → COMPLETED+enabled=0, recurrent otherwise → `next_run=croniter next` + PENDING + retry_count=0; dispatch by `isinstance(payload, ...)` branching; output truncated to `config.output_truncation_size`; if task missing at dispatch (re-fetch returns None) → log error, continue; accept: `_run_once()` test helper exposes single iteration; accept: missed oneshot → MISSED, recurrent restart → next_run recomputed.
- [ ] T6.2 Add `_run_once()` helper method to `SchedulerService` for deterministic test iteration — fetches one due task, dispatches it, returns; accept: callable from tests without starting full loop.
